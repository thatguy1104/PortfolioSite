The MoSCoW List:

Below you may find classification of the project's requirements

  • Must Have:

    • Reliable and secure backend structure based on AWS used to store user data
    • Users uniquely identified by a primary key to ensure data integrity
    • Secure authentication and communication system between the application and the server
    • Relevant websites/articles found using a web scraper. The web crawler will create an index of relevant links that will be displayed in the news section
    • Geolocation services to provide nearby events, gatherings, conferences, and earliest webinars
    • Available tasks will be stored in the database along with user data

  • Should Have:

    • Push notifications made possible using FCM (Fire Cloud Messaging)

  • Could Have:

    • A dictionary could be maintained wherein the activity is the key and the response displayed to the user is the value
    • User may log activities for the app to recommend an alternative activity that would leave less of a carbon footprint. Sentiment analysis to beperformed on the user input
    • Developers collect data over time, use data for analytical purposes, track challenge completion across the neighborhoods
    • Developers collaborate with members of the government to conduct nationwide analysis, with privacy implications in mind

  • Would Like to Have:

    • Introduce a point-reward system, whereby a user is rewarded points for achieving challenges, attending climate-change-related events, and staying informed by reading news articles on the aforementioned topic
